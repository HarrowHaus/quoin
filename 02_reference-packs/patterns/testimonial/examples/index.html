@quoin/pattern-testimonial — specimen

Three variants — compact, default, featured — × five states (default, empty, sparse, dense, loading, error) × seven microstates. Composes <avatar-stack> from vocab-app-shell for attribution.

Variant: default

Card-sized testimonial — fits two-up on desktop, stacks on mobile. Used in marketing 3-up sections.

3-column grid · density normal · token-grounded throughout · figure + blockquote + figcaption + cite semantic markup

Variant: compact

Chip-sized testimonial — fits 4–6 per row on desktop. Used in trust-bar sections under hero, or in pricing-page social-proof rails.

Variant: featured

Full-width pull-quote — section-anchor element. Display-tier type, generous padding, customer name + role at body scale. One per page; reserved for the strongest endorsement.

Multi-avatar attribution — team testimonial

Exercises the avatar-stack primitive's overflow indicator. Six contributors → 3 visible + "+3" overflow.

avatar-stack from vocab-app-shell · 3-visible cap · "+3" overflow indicator · role=group + aria-label exposes the full list to screen readers

Card-level microstates

Per-card microstates (subset shown — full set is default / hover / active / focus / focus-visible / disabled / loading / selected). Cards are tabbable when interactive (clickable cards open a case study) — keyboard focus + visible ring + Enter activates.

Accessibility checklist

  • Every card wraps a <blockquote> in a <figure> with the attribution in <figcaption> — semantically associated, screen-reader-correct.
  • Customer name uses <cite> (rendered without italic — cite is semantic, not stylistic).
  • Curly quotation marks via CSS quotes property + open-quote/close-quote generated content. Don't author curly quotes in source — let the typeface deliver them.
  • Star ratings (when present) carry aria-label=" out of 5 stars" on the rating element; individual stars are aria-hidden.
  • Multi-avatar groups use role="group" + aria-label listing the contributors; individual avatar circles are aria-hidden.
  • Interactive cards (clickable to open case study) are tabindex="0"; activate on Enter or Space.
  • Loading state: aria-busy="true" on the figure + skeleton animation respects prefers-reduced-motion.
  • Error state: role="alert" on the message container so screen readers announce immediately.
  • Tabular numerics on any score/rating ("9.4/10") so labels don't shift mid-render.

Typography precision

Quotes render via CSS quotes property — the actual blockquote text doesn't carry quote marks, the typeface delivers them. Curly apostrophes everywhere in copy ("don't", not "don't"). Em dashes for ranges. Hanging punctuation on prose paragraphs where supported.

Token-grounded throughout. The pattern's CSS reads from --surface-elevated, --border, --radius-card, --shadow-sm, --font-serif, --type-size-md and --leading-prose. Zero hardcoded values past the :root seed (which a real consumer replaces with their active token + theme pack).
