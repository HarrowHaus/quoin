Lifecycle — entering / visible / paused / exiting

Four lifecycle states a single toast moves through. entering: slide-in animation. visible: steady state with dismiss timer running (orange progress bar at bottom). paused: timer paused (hover or focus inside the toast). exiting: slide-out animation before DOM removal.

data-state="entering" Slide-in from edge, opacity 0 → 1.
data-state="visible" Steady — dismiss timer running (5s default).
data-state="paused" Hover or focus inside pauses timer (WCAG 2.2.1).
data-state="exiting" Slide-out, opacity 1 → 0, then DOM removal.

auto-dismiss vs persistent

Auto-dismiss toasts (info / success / warning, default 5s) carry a thin timer bar at the bottom that scaleX from 1 → 0 over the duration. Persistent toasts (critical errors) carry no timer + always show a close button.

auto-dismiss · 5s Default for info / success / warning. Visible timer at bottom.
auto-dismiss · 8s (longer) Custom duration via duration-ms attribute. Longer for toasts with an action button.
persistent · no timer Default for critical. User must dismiss explicitly.

Toast action variations

A toast has at most ONE action button. Multiple-action cases use modal-dialog instead. Action labels are imperative verbs (Undo / Retry / View / Open), never OK/Cancel.

Accessibility checklist

  • Region semantics: <div role="status" aria-live="polite" aria-atomic="false" aria-label="Notifications">. aria-atomic="false" tells AT to announce only the changed (newly added) content, not the whole region on every change.
  • Live levels: aria-live="polite" default. For critical-only regions, use aria-live="assertive". Consider TWO regions when mixing criticality — keep polite from being upgraded.
  • WCAG 2.2.1 Timing Adjustable: Hover AND focus PAUSE the dismiss timer. Not "add more time" — pause. Resume original duration on mouseleave + focus-out. Allows users with motor or cognitive impairments to read at their own pace.
  • Persistent variant for critical errors: Critical toasts default to persistent (no timer). User must dismiss explicitly via close button or action.
  • Toast itself has no specific role: The parent toast-region's aria-live handles announcement. Adding role="alert" on each toast can cause duplicate announcements.
  • Tone icon aria-hidden: Icon is aria-hidden='true'; text content carries meaning. WCAG 1.4.1 — colour and icon alone are not sufficient.
  • Action button labels: Imperative verb-first ('Undo' / 'Retry' / 'View' / 'Open'). Not 'OK' / 'Cancel'. The toast title carries context; the action names the verb.
  • Single action per toast: At most ONE action button per toast. Multiple-action cases use pattern-modal-dialog instead.
  • Close button labelling: aria-label="Dismiss notification" (or "Dismiss <toast-title>"). The ✕ glyph is aria-hidden.
  • Focus pauses timer: Focusing the action button or close button pauses the dismiss timer. Tab order: visible toasts → action → close → next toast.
  • Focus restoration: On dismiss-by-close, return focus to the previous focusable. If the user just hovered, no focus change needed.
  • Stack limit: Cap visible toasts at 5 (max-visible attribute). When exceeded, oldest auto-dismisses to make room.
  • Reduced motion: Slide-in / slide-out transitions + timer-bar animation collapse to instant under prefers-reduced-motion: reduce. No popping motion for sensitive users.
