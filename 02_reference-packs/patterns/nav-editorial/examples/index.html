pattern-nav-editorial — specimen

Magazine / editorial-style top navigation. Distinct from marketing nav (heavily CTA-driven) and docs nav (sidebar tree). The editorial convention foregrounds the masthead, presents categories as a strong horizontal rule, and keeps utility actions in a clearly-separated row. Eight primitives across three registers — stacked (canonical newspaper layout), single-row (compressed), centered-masthead.

Accessibility checklist

  • Banner landmark: <header role="banner"> — only one per page.
  • Sections nav landmark: Categories list wrapped in <nav aria-label="Sections">. Distinct from utility nav.
  • Utility cluster role: <div role="group" aria-label="Utility actions"> wraps account/saved/search/subscribe.
  • aria-current="page": Active category gets aria-current="page" + visible 3px underline indicator + bold weight.
  • Masthead not a heading: Wordmark is <a>, not <h1>. Editorial pages reserve <h1> for the article headline.
  • Edition date semantics: Edition date is plain <p> (not a heading) — descriptive text, not part of the document outline.
  • Search landmark: <form role="search"> identifies the search field as a landmark. Input has visually-hidden <label>.
  • Search expand toggle: Icon button has aria-expanded + aria-controls + state-paired aria-label ("Expand search" / "Close search").
  • Mobile toggle: aria-expanded + aria-controls + state-paired aria-label ("Open menu" / "Close menu").
  • External brand link: Masthead <a aria-label="..."> names the destination clearly (home, not "logo").
  • Focus styles: Every interactive element receives visible :focus-visible outline via --focus-ring tokens.
  • Reduced motion: Search expand width transition collapses to instant under prefers-reduced-motion: reduce.
  • Subscribe-pill copy: Imperative verb ("Subscribe" / "Subscribe now"). Not "Click here" / "Sign up" / "Learn more".
