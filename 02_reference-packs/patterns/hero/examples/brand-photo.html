Photo-led marketing hero — the brand-photo variant of @quoin/pattern-hero. Inherits the 6 mandatory slots; activates two conditional slots — hero-media (responsive <picture>) and hero-overlay (text-legibility scrim). Four layouts × four overlay kinds × three strengths. LCP optimization via fetchpriority="high" + AVIF/WebP/JPEG source sets. data-layout is an orthogonal axis to data-alignment (Cons. 3 Q3).

Responsive <picture> + LCP optimization contract

Production HTML for the hero image. AVIF first (smallest, modern), WebP next (broad support), JPEG fallback. fetchpriority="high" tells the browser this is the LCP candidate. decoding="async" avoids blocking text rendering.

<picture data-pattern="hero-media" data-aspect="4:3" data-crop="cover" data-focal-point="center">
  <!-- AVIF: smallest payload, modern browsers -->
  <source type="image/avif"
          media="(min-width: 800px)"
          srcset="hero-desktop.avif 1x, hero-desktop@2x.avif 2x">
  <source type="image/avif"
          srcset="hero-mobile.avif 1x, hero-mobile@2x.avif 2x">

  <!-- WebP: broader support -->
  <source type="image/webp"
          media="(min-width: 800px)"
          srcset="hero-desktop.webp 1x, hero-desktop@2x.webp 2x">
  <source type="image/webp"
          srcset="hero-mobile.webp 1x, hero-mobile@2x.webp 2x">

  <!-- JPEG fallback + LCP optimization -->
  <img src="hero-desktop.jpg"
       srcset="hero-mobile.jpg 800w, hero-desktop.jpg 1600w"
       sizes="(min-width: 800px) 50vw, 100vw"
       alt="Designer's hand drawing on a tablet, Galley Easel UI visible"
       width="800" height="600"
       fetchpriority="high"
       decoding="async">
</picture>

Accessibility checklist

  • Single h1 per page: hero-headline uses <h1>.
  • aria-labelledby: Section points to the h1's id.
  • WCAG 1.1.1 (Alt-text): If photo is informative, <img alt="descriptive-text">. If decorative, <img alt=""> (empty alt — NOT aria-hidden). Empty alt='' is the correct way to mark a decorative image; AT skips it.
  • WCAG 1.4.3 (Contrast): On image-overlay layouts, headline + subhead promote to --text-on-accent (white) or --text-emphasis (near-black) based on the underlying overlay tone. Overlay register selection guarantees ≥4.5:1 contrast on body text and ≥3:1 on large text.
  • Overlay is decorative: hero-overlay always carries aria-hidden="true". The image's alt-text carries semantic meaning.
  • LCP optimization: Hero image uses fetchpriority="high" + decoding="async". Do NOT use loading="lazy" — that's wrong for above-fold images.
  • Responsive sources: Use <picture> with AVIF → WebP → JPEG cascade. Different crops at different breakpoints via <source media="..."> for art direction.
  • Focal-point preservation: object-position set via data-focal-point attribute. Critical for portraits where the face must stay in frame across responsive crops.
  • Ghost CTA on image: Backdrop-filter blur + semi-transparent surface for legibility on unpredictable image content. Same technique as hero-gradient-mesh.
  • External resource (image) is host-provided: This pack ships ONLY the layout + responsive contract. Production photography is licensed and supplied by the host application — never bundled with the pack to avoid licensing entanglement.
  • Width + height attributes on img: Always specify width + height on the <img> to reserve layout space and prevent CLS (Cumulative Layout Shift).
