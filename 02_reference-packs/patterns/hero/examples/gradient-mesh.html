Why OKLCH

OKLCH (CSS Color Module Level 4) gives perceptually-uniform color interpolation. RGB and HSL gradients between vibrant colors produce muddy gray midpoints; OKLCH keeps the chroma path clean. Baseline browser support since Safari 15.4 / Firefox 113 / Chrome 111.

/* Cool palette — 3 radial-gradient layers in OKLCH */
[data-pattern="hero-background"] {
  background:
    radial-gradient(circle at 20% 30%, oklch(85% 0.12 240) 0%, transparent 50%),
    radial-gradient(circle at 80% 20%, oklch(82% 0.14 200) 0%, transparent 55%),
    radial-gradient(circle at 50% 90%, oklch(85% 0.10 270) 0%, transparent 60%),
    oklch(98% 0 0);
}

/* @supports fallback for browsers without OKLCH (pre-2022 Safari, etc.) */
@supports not (color: oklch(0% 0 0)) {
  [data-pattern="hero-background"] {
    background: var(--accent-recede) !important;
  }
}

Accessibility checklist

  • Single h1 per page: hero-headline uses <h1>. Same single-h1 discipline.
  • aria-labelledby: Section points to the h1's id.
  • Background is decorative: hero-background always carries aria-hidden="true".
  • WCAG 1.4.3 (Contrast): Headline uses --text-emphasis (15% lightness) — 12:1 ratio against the lightest mesh point (85% lightness). Body subhead defaults to --text; dramatic intensity promotes to --text-emphasis. Optional data-background-tint="subtle" on subhead adds a soft surface underlay for guaranteed legibility on busy mesh backgrounds.
  • Ghost CTA legibility: On mesh backgrounds, ghost-intent CTAs get backdrop-filter: blur(8px) + semi-transparent surface to stay legible against blob colors.
  • OKLCH fallback via @supports: Browsers without OKLCH support (pre-2022) fall back to a solid --accent-recede background via @supports not (color: oklch(0% 0 0)). The hero still works; the visual just isn't the mesh.
  • WCAG 2.3.3 (Reduced motion): Mesh is static — no animated movement by default. shift-on-hover is opt-in and is suppressed via the global * { animation: none !important; } reduced-motion rule.
  • Grain overlay aria-hidden: The SVG noise overlay is purely decorative via ::after pseudo-content (not in the accessibility tree).
  • Trust-badge images: Same alt-text discipline as hero-type-only when hero-meta uses trust-badges register.
  • Focus styles: CTAs receive visible :focus-visible outline via --focus-ring tokens.
