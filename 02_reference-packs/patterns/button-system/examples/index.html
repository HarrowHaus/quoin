Pattern-level states

The button system declares 4 pattern-level states (in addition to the 8 microstates per button). These describe the entire button cluster, not individual buttons.

state: default

Normal interactive state — buttons respond to input.

state: loading (form submission in progress)

Primary action spins; secondary actions disable until response.

state: success (post-action confirmation)

Primary shows momentary success affordance; live region announces. Reverts to default after ~1500ms in production.

state: error (form validation failed)

Primary remains active to allow retry; error message appears in the surrounding form (not in the button itself — buttons don't carry error state, the form does).

Accessibility checklist

  • Every action-button renders as a semantic <button> by default; <a> when href is supplied.
  • Icon-only buttons require aria-label — enforced at compile time.
  • Loading state announces via aria-busy="true" + aria-live="polite".
  • Selected/toggle buttons use aria-pressed; segmented-control buttons use role="radio" + aria-checked.
  • Disabled state uses [disabled] for skip-focus or aria-disabled="true" for retain-focus + announce-as-dimmed; consumer chooses per context.
  • Focus ring honors --focus-ring-width + --focus-ring-offset + --focus-ring tokens (visible only via :focus-visible — mouse clicks don't trigger ring).
  • 44×44 minimum hit target on sm/md/lg sizes; xs is reserved for chrome contexts with parent-row hit areas.
  • Reduced-motion fallback: transitions collapse to 0ms; loading spinner stops animating; CSS @media (prefers-reduced-motion: reduce) handles globally.
  • Text + icon convey state; never color alone (e.g., success state uses checkmark + label, not just colour swap).

Typography precision

Labels render with font-variant-numeric: tabular-nums so counts and percentages in labels (e.g., "Save (3 unsaved)") don't shift the layout. Curly apostrophes used in copy: "Don't save" not "Don't save". Em dashes for ranges. Ellipsis character (…) for in-progress affordances ("Saving…", "Loading…").
