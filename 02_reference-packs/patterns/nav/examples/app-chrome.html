Accessibility checklist

  • Banner landmark: <header role="banner"> — only one per page.
  • App-shell nav: <nav aria-label="App shell"> distinguishes from in-page sub-nav and page-tabs nav.
  • Cmd+K shortcut advertised: aria-keyshortcuts="Meta+K Control+K" on the search trigger; aria-haspopup="dialog" because the palette is a modal dialog.
  • Popover ARIA pattern: Every popover trigger uses aria-haspopup="true" + aria-expanded="true|false" + aria-controls="". Popover has aria-labelledby pointing back to trigger.
  • Notification badge: Trigger's aria-label includes the count ("Notifications, 3 unread"); the visual badge digit is aria-hidden because the label already says it.
  • User menu name: Trigger's aria-label includes the user's display name ("Account menu — Elena Park").
  • Breadcrumb: <nav aria-label="Breadcrumb"> wraps an <ol>; last segment is a <span aria-current="page"> (not a link).
  • Page tabs: <nav aria-label="Page sections">; active tab carries aria-current="page" since each tab is its own route.
  • Status banner: Info + warning use role="status" + aria-live="polite". Critical uses role="alert" + aria-live="assertive".
  • Offline state: Sync-dependent actions become aria-disabled="true" with explicit aria-label noting the reason ("Notifications unavailable while offline").
  • Focus management: Every popover Escape closes + returns focus to trigger. Click-outside also closes. Cmd+K palette traps focus while open.
  • Focus styles: Every interactive element receives visible :focus-visible outline via --focus-ring tokens.
  • Reduced motion: Pulse animation on status-banner dot collapses to none under prefers-reduced-motion: reduce.
