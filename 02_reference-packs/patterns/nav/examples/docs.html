Accessibility checklist

  • WCAG 2.4.1 (Bypass Blocks): Skip-to-content link as first focusable element. Visually hidden until focused. Points to id="main" on the host page's main content wrapper.
  • WCAG 2.4.6 (Headings and Labels): Top-bar nav has aria-label="Top", sidebar has aria-label="Documentation", meta-actions cluster has role="group" aria-label="Documentation actions". Distinct nav landmarks.
  • WCAG 4.1.2 (Name, Role, Value): Active sidebar link gets aria-current="page". Collapsible section headings have aria-expanded="true|false" + aria-controls.
  • Search ARIA: aria-keyshortcuts="Meta+K Control+K" + aria-haspopup="dialog" + aria-expanded. Same contract as nav-app-chrome's app-global-search.
  • Mobile toggle: aria-expanded + aria-controls="docs-sidebar" + state-paired aria-label ("Open documentation menu" / "Close documentation menu").
  • Mobile drawer: When open, sidebar gets aria-modal="true" + focus trap + Escape-to-close + body scroll lock.
  • External link affordance: GitHub link carries rel="noopener noreferrer" + aria-label noting new-tab destination.
  • Sidebar tree depth indicators: data-depth attribute on links signals visual indentation. Does NOT change the semantic structure — the flat <ul> remains accessible.
  • Sidebar section collapse: Heading text is wrapped in <button aria-expanded> so keyboard users can collapse/expand. Enter+Space activate.
  • Badge labels: NEW/BETA/DEPRECATED badges carry aria-label describing the badge meaning. Visual letters are aria-hidden.
  • Theme switcher: Tri-state (light/dark/auto). Implemented as aria-pressed on the button, with aria-label naming the current state.
  • Focus styles: Every interactive element receives visible :focus-visible outline via --focus-ring tokens.
  • Reduced motion: Sidebar drawer slide-in collapses to instant under prefers-reduced-motion: reduce.
