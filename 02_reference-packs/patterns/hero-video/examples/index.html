⚠️ WCAG 2.2.2 (Pause, Stop, Hide) — Any moving content that starts automatically AND lasts more than 5 seconds AND is presented in parallel with other content MUST have a user-controllable pause mechanism. The hero-video-controls primitive is part of the pattern's hard contract — it is not optional. Auto-playing background video without a visible pause control is a WCAG 2.2.2 violation.
📹 Video-licensing note — This specimen uses CSS-animated gradient placeholder fixtures (labelled "PLACEHOLDER FIXTURE — animated stand-in for production video") in place of real video files. The pattern does NOT bundle video assets; production hosts must supply their own licensed video via the <source> elements. The pattern ships only the layout, <video> contract, controls primitive, and overlay scaffolding.

Register: background-loop · default

Video fills the hero corner-to-corner. Muted, autoplay, loop, playsinline. Gradient overlay for text contrast. The WCAG 2.2.2 pause control sits bottom-right. Click the control to toggle the pattern between data-state="playing" and data-state="paused".

State: paused — user-controlled OR reduced-motion default

When data-state="paused", the video stops, the control's aria-label becomes "Play hero video", and the icon switches to a play glyph. Under prefers-reduced-motion: reduce, the section starts in this state automatically (host JS handles the detection).

GALLEY · PAUSED STATE

Video paused.

User pressed pause, or the user's OS reports prefers-reduced-motion. The poster image (or last frame) fills the video bounds.

Production <video> contract

The HTML that ships in production. WebM (vp9) primary; MP4 (h.264) fallback. autoplay muted loop playsinline preload="metadata" for background-loop register. Poster image is the LCP candidate. Inner <img> fallback for very-old browsers with no <video> support.

<video data-pattern="hero-video-media"
       autoplay
       muted
       loop
       playsinline
       preload="metadata"
       poster="hero-poster.jpg"
       aria-label="Time-lapse of designers using Galley Easel">
  <source src="hero.webm" type="video/webm">
  <source src="hero.mp4" type="video/mp4">

  <!-- Fallback for browsers without <video> support -->
  <img src="hero-poster.jpg"
       alt="Time-lapse of designers using Galley Easel"
       width="1920" height="1080"
       fetchpriority="high">
</video>

<!-- WCAG 2.2.2 — REQUIRED pause control -->
<button data-pattern="hero-video-controls"
        data-position="bottom-right"
        type="button"
        aria-label="Pause hero video">
  <icon name="pause" aria-hidden="true" />
  <span>Pause</span>
</button>

<script>
  // Host JS: detect reduced-motion preference
  if (matchMedia('(prefers-reduced-motion: reduce)').matches) {
    document.querySelector('[data-pattern="hero-video-media"]').pause();
    document.querySelector('[data-pattern="hero-video-section"]').dataset.state = 'paused';
  }
</script>

Accessibility checklist

  • WCAG 2.2.2 (Pause, Stop, Hide) — HARD CONTRACT: The hero-video-controls primitive is REQUIRED. Auto-playing background video over 5 seconds without a pause control is a violation. The control is always visible (not on-hover-only).
  • Pause-control aria-label state-aware: Switches between "Pause hero video" (when playing) and "Play hero video" (when paused). Updated on every toggle.
  • Reduced-motion default-pause: Host JS detects prefers-reduced-motion: reduce on mount and starts the video paused. The poster fills the video bounds; user must press play to start.
  • Muted is mandatory: <video muted> for background-loop register. Browsers block autoplay with audio unless user explicitly grants permission.
  • playsinline is mandatory on iOS: Without playsinline, mobile Safari shows the video in fullscreen on play.
  • WCAG 1.1.1 (Alt-text): Poster image alt-text describes what the video shows (e.g., "Time-lapse of designers using Galley Easel"). Same text on the video's aria-label.
  • WCAG 1.4.3 (Contrast): Overlay primitive provides contrast guarantee. Headline + subhead default to --text-on-accent (white) on video.
  • Video fallback: <video> with inner <img> renders the poster on browsers without video support.
  • preload="metadata" not "auto": Reduces initial payload. Full video is fetched only when autoplay triggers.
  • Single h1 per page: hero-video-headline uses <h1>.
  • aria-labelledby: Section points to the h1's id.
  • External resource (video) is host-provided: This pack ships ONLY the layout + controls + responsive contract. Production video is licensed and supplied by the host.
  • Focus styles: Pause control + every CTA receive visible :focus-visible outline.
