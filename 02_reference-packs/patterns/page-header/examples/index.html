Accessibility checklist

  • No banner role: page-header is just a <header> WITHOUT role="banner". The banner landmark lives on the parent pattern-nav-app-chrome (only ONE banner per page).
  • Heading hierarchy: page-header-title uses <h1> by default (the page-header IS the page's first heading in most app contexts). Downshift to <h2> when embedded inside a marketing template that has its own <h1>.
  • Breadcrumb eyebrow: When eyebrow register='breadcrumb', the inner <nav aria-label="Breadcrumb"> + <ol> structure mirrors app-breadcrumb from nav-app-chrome.
  • Current page marker: Breadcrumb last segment uses <span aria-current="page"> (not a link).
  • Visual separators: · / / separators are aria-hidden="true".
  • Avatar stack composed from vocab-app-shell: Inherits its aria-label from the primitive ("Owned by Elena Park" / "Shared with 4 people").
  • Action cluster role: <div role="group" aria-label="Page actions"> wraps the cluster.
  • Icon-only buttons: Must have explicit aria-label naming the action.
  • Loading state: data-state="loading" + aria-busy="true". Skeleton shimmer collapses under prefers-reduced-motion.
  • Error state: Title text wraps in <span role="alert"> so AT announces on first paint.
  • Text wrapping: page-header-title uses text-wrap: balance; page-header-subtitle uses text-wrap: pretty.
  • Editable titles: If the title is inline-editable (e.g., document name in Galley Notes), the host page may set contenteditable + role="textbox" + aria-label="Document title". The behaviour is host-provided in v1.
