Composition lineage — what footer-mega consumes

The newsletter primitive composes sibling pattern packs rather than re-implementing inputs and buttons. The brand-block's optional locale-picker also composes a button. Documented for legibility.

Consumed primitive Source pack Used in How this pattern uses it
<form-control type="email"> @quoin/pattern-form-fields footer-newsletter Email input with validation. Inherits 33 tokens + 8 microstates + 4 field-level states. [aria-invalid] handled by form-fields; this pattern just sets data-state on the parent form.
<action-button intent="primary"> @quoin/pattern-button-system footer-newsletter submit Submit button. [aria-busy] microstate for submitting state delegated entirely to action-button.
<action-button intent="ghost"> @quoin/pattern-button-system footer-brand-block locale-picker (optional) Ghost-intent button that opens a locale popover. Popover content out of scope for v1 — host page provides.
canonical tokens @quoin/tokens-baseline every CSS value Surfaces / typography / borders / motion / shadows. Footer aesthetic flips with theme pack swaps; structure is invariant.

Accessibility checklist

  • Landmark: <footer role="contentinfo"> — only one per page; serves as the canonical Page Footer landmark.
  • Footer nav: <nav aria-label="Footer navigation"> wraps the link grid, distinguishing it from the page's primary <nav>.
  • Social nav: <ul aria-label="Social media"> labels the sublist; each icon-only link has aria-label naming the destination platform (e.g. "GALLEY on GitHub").
  • Legal nav: <ul aria-label="Legal"> distinguishes legal links from main footer nav.
  • Current page marker: Links pointing to the active page get aria-current="page" with visible underline.
  • External link affordance: External links carry rel="noopener noreferrer" and optional aria-label noting "opens in new tab" when target="_blank".
  • Newsletter form label: <form aria-label="Newsletter signup"> distinguishes the inline form. The email input has its own <label for> connection.
  • Newsletter status announcements: aria-live="polite" for success; aria-live="assertive" for errors. Live region in DOM at all states (empty when no message).
  • Back-to-top: aria-label="Back to top" with focus management — on activation, focuses the page's first focusable element rather than just scrolling.
  • Focus styles: All interactive elements receive visible :focus-visible outline via --focus-ring tokens.
  • Heading hierarchy: Footer link-group headings default to <h3>; downshift to <h4> if the page uses <h2> for section heads.
  • Reduced motion: Inherits the reduced-motion handling from tokens (motion durations collapse to 0ms when prefers-reduced-motion: reduce).
