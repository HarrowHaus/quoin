Tone × direction — explicit, not auto-derived

For most metrics, "up = positive" — but for spend / cost / latency / error rate, "up = negative". The tone attribute is explicit per metric. Example: Monthly recurring went UP by $24 — direction=up, but tone=negative because more spend is bad. Avg cost per service went DOWN by $1.20 — direction=down, tone=positive because lower average is good.

Composition lineage — what stat-card consumes

Consumed primitiveSource packUsed inHow this pattern uses it
<action-button intent="ghost"> @quoin/pattern-button-system Optional in-card actions, info-tip triggers Ghost buttons for inline actions (e.g., "View detail" link inside the card, or the info-tip "?" trigger on stat-label).
canonical tokens @quoin/tokens-baseline every CSS value Surfaces / typography / borders / motion / shadows / focus-ring. Delta tones (--success / --critical / --text-recede) and their *-recede backgrounds drive the three delta tones.
Composition decision documented: stat-card does NOT compose validation-status-icon from pattern-form-validation for delta indicators, despite some semantic overlap (success / critical tones). The icon vocabularies are different — validation icons are check / x / warning / spinner; delta indicators are up-arrow / down-arrow / flat. Visual literacy is clearer when these vocabularies stay separate.

Accessibility checklist

  • Visually-hidden full value: Visual unit symbols ($ / % / GB / /mo) are aria-hidden="true". A visually-hidden span announces the full value with units spelled out ("$310.18 per month") for AT.
  • Tabular nums: font-variant-numeric: tabular-nums on stat-value ensures digits align column-wise when multiple stat-cards stack — critical for visual scanning across a KPI row.
  • Delta tone explicit, not derived: Tone (positive / negative / neutral) is set per metric. "Up = positive" doesn't generalise — for spend / cost / latency, "up = negative". The primitive renders the visual; the host sets the semantic.
  • Delta accessible name: aria-label="+$24 this month — higher spend" on stat-delta, OR <span class="sr-only">up</span> alongside the arrow SVG.
  • Arrow aria-hidden: Arrow SVGs in stat-delta are aria-hidden="true" — magnitude + period text carry the meaning per WCAG 1.4.1.
  • Sparkline accessibility: <svg role="img" aria-labelledby="..."> with inline <title> + <desc> gives AT a textual description of the trend.
  • Loading state: data-state="loading" + aria-busy="true" on the card. Skeleton shimmer collapses to none under prefers-reduced-motion.
  • Error state: stat-context wraps in role="alert" so AT announces "Couldn't load — retry" on first paint.
  • No-data state: Value renders as em dash (—). stat-value carries aria-label naming the absence ("Monthly recurring: no data available").
  • Interactive card semantics: Whole-card click uses <a> or <button type="button"> as the outer element. Inner duplicate links are removed or aria-hidden to avoid double tab stops.
  • Focus styles: Interactive cards receive :focus-visible outline via --focus-ring tokens.
  • Composition decision: Does NOT compose validation-status-icon — icon vocabularies are different (validation = check/x/warning; delta = arrows).
