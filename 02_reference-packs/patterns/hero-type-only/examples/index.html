<action-button intent="ghost" / "secondary"> @quoin/pattern-button-system hero-actions (secondary CTAs) De-escalated CTAs alongside the primary — View pricing / Watch demo / Talk to sales.
canonical tokens @quoin/tokens-baseline every CSS value Typography (display + sans), spacing scale, surfaces (5 background variants), motion. The dark background variant inverts text + accent + ghost via inline overrides.
Hero category: hero-type-only is 1 of 5 hero variants in the catalog. P1 lands the remaining 4 (hero-animated, hero-gradient-mesh, hero-brand-photo, hero-video) at the front of P1 to recover the marketing-surface hole left by the de facto P0 pivot to application-surface patterns.

Accessibility checklist

  • Single h1 per page: hero-headline uses <h1>. Marketing pages should have exactly ONE h1 — and it lives in the hero. Subsequent sections use h2.
  • aria-labelledby: <section aria-labelledby="hero-headline-id"> gives the hero a programmatic name pointing to its h1.
  • Eyebrow is NOT a heading: hero-eyebrow renders as <p> (not h2/h3) — marking it as a heading would break the page outline. The h1 IS the page anchor.
  • text-wrap balance + pretty: hero-headline uses text-wrap: balance for visually-balanced multi-line headlines; hero-subhead uses text-wrap: pretty for orphan-avoidance.
  • Action cluster role: <div role="group" aria-label="..."> wraps the cluster. Label names the purpose ("Sign up actions" / "Hero actions" / "Try Dues actions").
  • Hero CTA size discipline: Hero CTAs default to button size="md"; use size="lg" only for oversized hero size. Don't mix sizes within a cluster.
  • Trust-badge alt-text: Trust-badge images carry alt-text naming the certification ("SOC 2 Type II compliant"). Don't use aria-hidden — these are meaningful credentials.
  • Social-proof verifiability: Social-proof claims should be verifiable real numbers. If approximate, indicate ("over 14,000" not "14,000").
  • Dark variant text contrast: The dark background variant overrides text colours to maintain contrast — text-on-accent foreground, oklch(80%) for subhead, oklch(70%) for meta + eyebrow.
  • Reduced motion: Hero is mostly static. Any future hover/scroll animation must respect prefers-reduced-motion: reduce.
