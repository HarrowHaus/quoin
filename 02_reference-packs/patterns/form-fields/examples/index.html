Accessibility checklist

  • Every <input> / <textarea> / <select> has a wired <label for>. No floating labels without proper aria-labelledby.
  • Help text linked via aria-describedby. Error messages additionally carry role="alert" + aria-live="polite" so screen readers announce dynamically.
  • Required fields marked with required attribute (native) AND visible * labelled with aria-label="required".
  • Field-level errors set aria-invalid="true" on the input. Form-level summary appears in role="alert" with anchor links to failed fields per WCAG 2.4.6 + 3.3.1 + 3.3.3.
  • Validating state uses aria-busy="true" + aria-live="polite" for async feedback.
  • Radio groups wrap in <fieldset> with <legend>. Don't use a generic <div> + label for groups.
  • Switch uses role="switch" + aria-checked. Not aria-pressed (that's for toggle buttons).
  • Indeterminate checkbox sets aria-checked="mixed" AND indeterminate property.
  • File input: the custom-styled wrapper is the visual surface; the native <input type="file"> is absolutely-positioned over it with opacity: 0 so it receives clicks + retains focus order. The wrapper itself is a <label> linked to the input so a click anywhere opens the picker.
  • Combobox follows WAI-ARIA 1.3 combobox pattern: role="combobox" + aria-expanded + aria-controls + aria-activedescendant + aria-autocomplete="list". Listbox has role="listbox"; options have role="option" + aria-selected.
  • Submit button shows aria-busy="true" during submission; live region announces state transitions.
  • 44×44 hit target on md + lg sizes (40px input height + 4px space-1 padding above/below brings it to ≥44px). Sm size is reserved for chrome / table-row contexts.
  • Focus rings use --focus-ring tokens, visible only via :focus-visible (mouse-clicks don't trigger the ring).
  • Reduced-motion: spinner stops, focus transitions become instant. @media (prefers-reduced-motion: reduce) handled globally.

Typography precision

Curly apostrophes throughout ("you'll", not "you'll"). Em dashes for in-line clarifications. Tabular numerics on every input + select + button label so digit-bearing values (postal codes, prices, seat counts, dates) don't shift mid-render. Mono font for addons (https://, .harrow.haus) and reveal-button labels (SHOW / HIDE) keeps them visually distinct from prose-tone labels.

Token-grounded throughout. Every CSS value reads from a --var. The :root seed at the top of this file is what a real consumer's tokens-baseline + active aesthetic pack supplies via the compiler pipeline. Zero hardcoded values past that seed.
