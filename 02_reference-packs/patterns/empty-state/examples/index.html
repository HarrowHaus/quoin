pattern-empty-state — specimen

Reusable empty-state primitive promoted from the data-table empty cell. Four variants (empty / filtered-empty / error / forbidden) × three sizes (sm / md / lg). Used anywhere a container has no content — list views, inboxes, search results, gallery grids, settings sub-pages, sidebar panels.

Accessibility checklist

  • Region landmark: <div role="region" aria-labelledby="..."> — gives the empty-state a named landmark.
  • Error-variant announcement: Error heading wraps in <span role="alert"> so AT announces "Couldn't load…" on first paint.
  • Filtered-empty live region: Parent container should have aria-live="polite" so AT announces "No matches" when the filter applies dynamically.
  • Illustration aria-hidden: Decorative icons / illustrations are aria-hidden="true". Heading + body carry the message per WCAG 1.4.1.
  • Heading hierarchy: Default <h3> at md/sm; <h2> at lg (full-page).
  • Action cluster role: <div role="group" aria-label="Actions"> wraps the cluster.
  • Constructive copy: Body acknowledges absence and suggests next action ("12 subscriptions total — try a different filter" beats "No matches").
  • Specific error context: Error body names the failure specifically ("Couldn't reach galley.harrow.haus") rather than generic "Something went wrong".
  • External-link affordance: Status / docs links open with rel="noopener".
