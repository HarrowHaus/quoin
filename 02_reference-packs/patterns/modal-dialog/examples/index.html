pattern-modal-dialog — specimen

Modal dialogs — alert (destructive confirmation), form (input collection), panel (rich content), command (Cmd+K palette). Focus trap, body scroll lock, inert background, Escape + backdrop dismiss, focus restoration to trigger on close. Six primitives. Composes @quoin/pattern-button-system for action buttons and @quoin/pattern-form-fields for form-variant dialogs.

Footer registers — 4 patterns

primary-only (informational), cancel-plus-primary (the 80% recommendation), destructive (delete confirmations — critical-intent right-most), extended (tertiary + cancel + primary). Cancel buttons live LEFT visually but should be FIRST in tab order.

Accessibility checklist

  • Dialog role: role="dialog" for most variants; role="alertdialog" for alert variant. aria-modal="true" tells AT siblings outside dialog are inaccessible.
  • Dialog naming: aria-labelledby="..." required, pointing to heading inside modal-header. aria-describedby="..." optional, especially important for alertdialog.
  • Inert background: Every sibling of the dialog's root gets the inert HTML attribute when dialog is open. Removes them from accessibility tree AND makes non-interactive. Polyfill: aria-hidden="true" + tabindex="-1" on focusable descendants.
  • Body scroll lock: overflow: hidden on <body> (or <html>) while dialog is open. Restore on close. Prevents accidental background scroll on touch devices.
  • Focus trap: On open, focus moves to first focusable inside dialog. Tab cycles through dialog focusables; Tab from last → first; Shift+Tab from first → last.
  • Focus restoration: On close, focus returns to the element that triggered the dialog. Store trigger in JS state.
  • Escape closes: Escape key closes most variants. For alertdialog where dismissal is intentional, MAY suppress — document the deviation.
  • Backdrop click dismisses: Click on backdrop closes most variants. For alertdialog, MAY suppress — but close button in header still works.
  • Cancel button tab order: Visually leftmost (in cancel-plus-primary register) but FIRST in tab order. Keyboard users tab into Cancel and must tab forward to reach destructive primary action.
  • Destructive button text: Names the consequence explicitly ("Remove subscriptions" not "Confirm"). Guards against accidental destructive activation.
  • Async submit state: Submit button gets aria-busy="true". Cancel button gets aria-disabled="true" to prevent dismissal mid-submission.
  • Command palette ARIA: role="dialog" + aria-haspopup="dialog" on trigger; aria-keyshortcuts="Meta+K Control+K" on trigger. Results list uses role="listbox" + role="option" with aria-selected; aria-activedescendant on the input announces selected result without moving DOM focus.
  • Close button labelling: aria-label="Close dialog" (or "Close <dialog-title>"). The ✕ glyph itself is aria-hidden.
  • Reduced motion: Fade-in / fade-out transitions collapse to instant under prefers-reduced-motion: reduce. No popping-in animation for users sensitive to motion.
