Reduced motion compliance

Under @media (prefers-reduced-motion: reduce), ALL entrance animations collapse to a single 120ms opacity fade. Stagger delays collapse to 0. Ambient accents (pulse, drift) are hidden via display: none. Scroll-parallax mode disables the scroll listener.

@media (prefers-reduced-motion: reduce) {
  :root {
    --motion-fast: 0ms;
    --motion-normal: 0ms;
    --transition-default: none;
    --duration-entrance: 120ms;
    --stagger-step: 0ms;
  }
  * { animation: none !important; }

  [data-pattern="hero-animated-section"] * {
    transform: none !important;
    animation: opacity 120ms ease-out !important;
  }

  /* Decorative motion stops entirely — not slowed, stopped. */
  [data-pattern="hero-animated-accent"] { display: none !important; }
}

Accessibility checklist

  • Single h1 per page: hero-animated-headline uses <h1>. Same single-h1 discipline as hero-type-only.
  • aria-labelledby: <section aria-labelledby="..."> points to the h1's id.
  • WCAG 2.3.3 (Animation from Interactions): ALL entrance animations honour prefers-reduced-motion: reduce. Under reduce, motion collapses to a 120ms opacity fade. Stagger delays become 0. Ambient accents disappear via display: none.
  • No aria-live on entrance: The headline is in DOM from initial paint; the entrance animation is decoration. Don't add aria-live="polite" to the headline — AT announces via natural reading order.
  • Word/character spans are inline: [data-word] and [data-char] wrappers use display: inline-block. They don't break text selection or screen-reader continuity — AT reads the underlying text as a single word/sentence.
  • Decorative accents aria-hidden: hero-animated-accent always carries aria-hidden="true".
  • Trust-badge images: If hero-animated-meta uses trust-badges register, badge images carry alt-text naming the certification — not aria-hidden.
  • Caret blink under type-in: The blinking block caret ▎ is rendered as a CSS ::after with content: "▎". It's pseudo-content (not in the accessibility tree). Under reduced-motion, it's hidden entirely (the * { animation: none } rule disables the blink).
  • Scroll-parallax requires JS: Pure-CSS scroll-driven animations have limited browser support. v1 falls back to no parallax when JS is disabled. Host page provides the scrollY listener; the listener is skipped entirely under reduced-motion.
  • Focus styles: Every CTA receives visible :focus-visible outline via --focus-ring tokens. The entrance animation does NOT include any focus-state animation — focus is steady.
  • Single h1 + section role: The section element implicit role serves the document outline. No role="banner" — hero is content, not the page banner (page banner lives on nav-marketing or nav-app-chrome).
