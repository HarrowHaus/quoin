Quoin Spec Playground Packs Get started GitHub Home / Migrate / From shadcn/ui Migrate from shadcn/ui shadcn/ui ships React components you copy into your repo and own forever. Quoin ships semantic HTML elements you write in templates. Different unit of authorship; the same instinct that intent-bearing names beat utility strings. Quoin is not a React component library. It's a build-time compiler. Use it in any framework — React, Svelte, Astro, plain HTML. The compiler runs at build time and emits standard HTML. When to use which

Keep shadcn/ui for interactive React components that need state, focus management, or accessibility behavior — dialogs, command palettes, dropdown menus, comboboxes. Those are real React work.

Use Quoin for the surrounding page — layout, editorial chrome, marketing sections, forms, dashboards. Anything where the value of authoring is semantic clarity, not behavior.

The two compose. A Quoin source file can include a React island for an interactive shadcn component; the rest of the page is Quoin tags compiled at build time.

Theming

shadcn/ui ships CSS custom properties named --background, --foreground, --muted, etc. The @quoin/tokens-shadcn pack translates those onto Quoin's canonical namespace — --surface, --text, --surface-recessed. Drop in the pack and your shadcn theme keeps working.

npm install @quoin/tokens-shadcn @quoin/vocab-shadcn @quoin/vocab-editorial @quoin/impl-tailwind Hybrid: Quoin + shadcn React islands

Use Quoin for the page chrome; mount shadcn React components as islands where you need interactive behavior. Astro, Next.js, and Vite all support this pattern. The page's static shape is semantic and reviewable; the interactive bits stay React.
