Below the semantic layer, three other layers do the work the author never has to repeat.
  • 01
    Authoring Semantic tags. <authority-mark>, <reading-flow>, <primary-action>. The vocabulary lives in a vocabulary pack.
  • 02
    Tokens Three-layer DTCG tokens — base palette, semantic namespace, component overrides. The aesthetic lives in a token pack.
  • 03
    Compilation Build-time. Resolves semantic tags against the active packs, emits standard HTML + CSS. Zero runtime dependency.
  • 04
    Rendering The browser. Plain HTML and CSS. No proprietary engine, no shadow DOM, no Quoin runtime.

Colour palette, type scale, spacing, radius, motion. 30 packs — Material 3, Carbon, Polaris, Fluent, Geist, Radix, Tailwind… @quoin/vocab-* Semantic primitives. The named elements you author with. 11 packs — editorial, dashboard, essentials, marketing, docs, forms, shadcn, radix, headless, ark, daisy, mantine @quoin/impl-* Compilation target. How each primitive renders to HTML and CSS. 2 packs — impl-tailwind (Tailwind v4 classes), impl-raw-css (inline styles) The aesthetic is the swappable vendor. Intent is the durable contract. Intellectual lineage

    Christopher Alexander's A Pattern Language for the generative-grammar model. The Swiss / International Typographic Style and the brutalist editorial tradition for the naming and structural philosophy.

    DaisyUI for the prior-art proof that semantic compression works on top of Tailwind. Quoin extends the compression one layer higher — from class names to elements.

    Tailwind v4 for the three-tier token architecture exposed as native CSS custom properties. The W3C Design Tokens Community Group for the DTCG file format.

    Web Components for the tag-name conventions and the optional runtime fallback path. Build-time wins; the runtime exists for environments without a build step.
