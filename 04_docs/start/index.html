Quoin Spec Playground Packs Get started GitHub Home / Get started Get started Five minutes if you've used Tailwind. The flow: install the compiler, install one pack of each type, write a page, build. 1. Install

The compiler runs at build time. You'll also want one token pack, one vocabulary pack, and one implementation pack. The defaults below are a safe starting point — swap any of them later without changing your source.

npm install --save-dev @quoin/compiler npm install @quoin/tokens-baseline @quoin/vocab-editorial @quoin/impl-tailwind 2. Wire the Vite plugin

The plugin processes index.html and any other HTML file Vite picks up. It transforms every Quoin semantic element into standard HTML using your active pack stack.

// vite.config.ts import { defineConfig } from "vite"; import { quoin } from "@quoin/compiler/vite"; export default defineConfig({ plugins: [ quoin({ tokenPack: "@quoin/tokens-baseline", vocabularyPacks: ["@quoin/vocab-editorial"], implementationPack: "@quoin/impl-tailwind", projectTokens: { // Override any semantic token without forking a pack. // accent: "oklch(58% 0.18 250)", // "font-display": "Inter, system-ui, sans-serif" } }) ] }); 3. Write a page

Author in semantic elements. The compiler resolves each tag against the active vocabulary pack, applies your token values, and emits standard HTML and CSS at build time.

<!-- src/index.html --> <!doctype html> <html lang="en"> <head> <link rel="stylesheet" href="/tokens.css" /> <link rel="stylesheet" href="/styles.css" /> </head> <body> <canvas-block> <stack gap="loose"> <authority-mark>My first Quoin page</authority-mark> <lead-graf> A short intro. The headline and this paragraph are semantic elements; the compiler is doing the styling work. </lead-graf> <reading-flow> <p>Body prose lives inside reading-flow…</p> </reading-flow> <cluster> <primary-action>Save</primary-action> <secondary-action>Cancel</secondary-action> </cluster> </stack> </canvas-block> </body> </html> 4. Build npm run build

dist/index.html contains zero Quoin tags. The <authority-mark> compiled to an <h1>, <primary-action> compiled to a <button>, and Tailwind classes (or inline styles, if you used impl-raw-css) do the visual work.
