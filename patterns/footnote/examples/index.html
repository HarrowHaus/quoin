Variant: sidenote-margin (CSS Anchor Positioning)

The first variant uses CSS Anchor Positioning to render the footnote in the outer margin alongside the prose1

Tufte-style. The trigger anchors the content to the same vertical position via anchor-name + position-anchor. Baseline 2026.
. Wide-viewport readers see the note inline with the text; narrow viewports may want to fall back to footnote-bottom or popup-on-click.

Variant: footnote-bottom (classic register)

The default variant collects footnotes at the bottom of the document1. Works without any new platform features — useful for print fidelity and as a no-script fallback.

You can chain multiple2 footnotes inline.

  1. The classic 1500s register: footnotes collect at page bottom. ↩
  2. Even now, when most reading happens on screens, the bottom-of-block register remains the most reliable. ↩

Variant: popup-on-click (popover + anchor positioning)

The popup variant uses the platform popover attribute together with anchor positioning1

Click to reveal; click outside or press Esc to dismiss. No JS required — the browser's popover light dismiss handles it.
