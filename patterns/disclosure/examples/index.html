Single disclosure

The disclosure pattern toggles the visibility of a content region via a button. It's the foundational anatomy for accordions, FAQ sections, and "show more" affordances. The trigger's aria-expanded state mirrors the content's visibility; the trigger's aria-controls attribute references the content's id.

Multiple disclosures (manual accordion behavior)

Each disclosure operates independently. For "only-one-open" semantics, the operator's JS coordinates the group.

The native <details> element implements similar semantics but with a fixed disclosure-widget visual register. Quoin's disclosure pattern exposes the anatomy explicitly so aesthetic packs can fully style it. Use <details> when the native register is acceptable; use this pattern when you need full aesthetic control.
