Table-of-contents pattern

Introduction

This is the introduction. As you scroll, the TOC on the left updates the active item via IntersectionObserver. The current section gets a colored border-inline-start.

Try scrolling through the article — the active link reflects the heading currently in the top 30% of the viewport.

Anatomy

The pattern is built from a small set of slots. The container, the list, the items, and an optional indicator.

Slots

Mandatory: toc-container, toc-list, toc-item, active-indicator. Conditional: scroll-tracker, collapse-toggle.

Variants

Three positions (sticky-side / inline-top / floating-overlay) × three depths (h1-h2 / h1-h3 / h1-h6).

Active-section detection

IntersectionObserver watches the heading elements within the prose. The rootMargin defaults to '0px 0px -70% 0px' — a heading is considered active when it enters the top 30% of the viewport.

The script flips data-current and aria-current='location' on the matching toc-item. Styling responds via CSS attribute selectors.

Composition

This pattern declares prim-sequence as a peer pack — the toc-list slot is rendered via prim-sequence's ordered-list variant. No inlined ordered-list rendering lives in this pack.

Padding to allow scrolling past the last section.
