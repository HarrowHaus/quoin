Home Journal Work Home / Journal / The Bot Smell Problem The Bot Smell Problem For the last fifteen years, the dominant aesthetic of the web has been one of progressive simplification — flat surfaces, generous whitespace, and a vocabulary of utility classes wide enough to encode every visual decision twice over.

Quoin proposes a different layering. The author writes intent; the compiler resolves it; the browser receives standard HTML and CSS. Nothing about that flow is novel — it is the same pipeline as Tailwind, DaisyUI, and a generation of CSS frameworks. What is novel is the level of abstraction at which the author works.

A <primary-action> is not a button class string; it is a semantic role. Below the role, anything is possible: the same source compiles against a Swiss token pack or a brutalist one, and the rendered output is meaningfully different without the author touching the markup.
